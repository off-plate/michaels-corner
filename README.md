# Michael's Corner

Personal video creation hub. One website that holds all video work: ideas, scripts, status pipeline, per-platform checklists, set design reference, camera setup, brand.

Live site (after deploy): https://michaels-corner.netlify.app (or chosen domain)

## What's inside

- **Today** — at-a-glance dashboard (counts, upcoming, recent ideas)
- **Ideas** — quick capture, filter, promote to video
- **Videos** — kanban pipeline (idea → drafting → filming → editing → published)
- **Scripts** — full per-video editor (hook, outline, script, b-roll, CTA, thumbnail concept)
- **Checklists** — interactive per-video checklists for YouTube long, Shorts/TT/Reels, Instagram, LinkedIn. State saved per video.
- **Set Design** — locked-in 3/4 profile MCU reference (camera diagram, lighting, composition)
- **Camera Setup** — quick lookup card for settings
- **Brand** — colors, fonts, voice (editable)

## Stack

- Vanilla React via CDN + Babel standalone (no build for the user, but a tiny build step concatenates `src/*.jsx` into `index.html`)
- LocalStorage for persistence (Phase 1). Supabase wiring planned for Phase 2 — cross-device sync.
- Netlify hosting, GitHub source.

## Editing

Edit source in `src/*.jsx`. Then run:

```bash
./build.sh
```

This concatenates source files into `index.html`. Netlify serves `index.html`.

Working locally:

```bash
python3 -m http.server 8000
# open http://localhost:8000
```

## Folders

- `src/` — React components, one file per page
- `seed/` — markdown source-of-truth for content that pre-populated the app (kept for archival)
- `index.template.html` — HTML shell with all CSS
- `index.html` — generated by `build.sh`, do not edit directly
- `build.py` / `build.sh` — inliner
